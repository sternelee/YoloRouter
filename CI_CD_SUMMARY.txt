🔄 RELEASE WORKFLOW DETAILED
═══════════════════════════════════════════════════════════════════════════════

Step 1: Code Quality Check (check job)
  ├─ cargo test --lib --release
  ├─ cargo fmt --check
  └─ cargo clippy --release -- -D warnings
  
  Fails if:
    • Any test fails
    • Code format doesn't match rustfmt
    • Clippy finds warnings
  
  Duration: ~5-10 minutes
  Status: Must pass before proceeding

Step 2: Multi-Platform Build (build job)
  ├─ Linux x86_64
  │  └─ cargo build --release --target x86_64-unknown-linux-gnu
  │
  ├─ macOS x86_64
  │  └─ cargo build --release --target x86_64-apple-darwin
  │
  ├─ macOS ARM64
  │  └─ cargo build --release --target aarch64-apple-darwin
  │
  └─ Windows x86_64
     └─ cargo build --release --target x86_64-pc-windows-msvc
  
  Each platform:
    1. Downloads dependencies (cached)
    2. Compiles release binary
    3. Strips binary (Linux/macOS)
    4. Generates SHA256 checksum
    5. Uploads artifact
  
  Duration: ~5-15 minutes (parallel builds)
  Only runs if: check job passes

Step 3: Release Creation (release job)
  ├─ Downloads all build artifacts
  ├─ Generates comprehensive release notes
  │  ├─ Feature list
  │  ├─ Installation instructions
  │  ├─ System requirements
  │  └─ Download table with checksums
  │
  ├─ Creates GitHub Release
  │  ├─ Tag: v0.2.0
  │  ├─ Title: YoloRouter v0.2.0
  │  ├─ Body: Auto-generated release notes
  │  └─ Assets: All binaries + checksums
  │
  └─ Publishes to GitHub
  
  Duration: ~2-3 minutes
  Output: https://github.com/sternelee/YoloRouter/releases/tag/v0.2.0

Step 4: Documentation Publishing (publish-docs job)
  ├─ Builds documentation site
  │  ├─ README.md
  │  ├─ INSTALL.md
  │  ├─ USER_GUIDE.md
  │  └─ PROJECT_SUMMARY.md
  │
  ├─ Generates navigation
  ├─ Uploads to GitHub Pages artifact
  └─ Deploys to: https://sternelee.github.io/YoloRouter/
  
  Duration: ~2-3 minutes

🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Build Fails:
  1. Check GitHub Actions log
  2. Look for error message
  3. Fix locally and push
  4. Tag again if needed

Tests Fail:
  • Run: cargo test --lib --release
  • Fix and commit
  • Push to main

Format Issues:
  • Run: cargo fmt
  • Commit and push

Checksum Mismatch:
  • Re-download from release
  • Verify file size matches
  • Check internet connection

Binary Doesn't Work:
  • Check platform matches
  • Verify signature (if applicable)
  • Try from GitHub release page
  • Open GitHub Issue
